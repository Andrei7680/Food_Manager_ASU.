## 2. Краткое описание проекта
Проект представляет собой **автоматизированную систему управления (АСУ)** учётом продуктов питания в домохозяйстве. Система выполняет функции:
- Сбор и хранение данных о продуктах (аналог базы данных параметров технологического процесса)
- Автоматический контроль сроков годности (аналог мониторинга предельных значений)
- Визуализация состояния объектов управления (цветовая индикация: зелёный — норма, жёлтый — предупреждение, красный — авария)
- Формирование отчётности и статистики

Проект демонстрирует базовые принципы построения АСУ: сбор данных → обработка → визуализация → принятие решений.

## 3. Используемый стек технологий
| Компонент | Технология | Аналог в промышленной АСУ |
|-----------|------------|---------------------------|
| Бэкенд | Python 3.10 | Язык программирования ПЛК / SCADA |
| Веб-фреймворк | Flask | Веб-сервер АРМ оператора |
| База данных | SQLite | Архив параметров (тренды) |
| ORM | SQLAlchemy | Промышленные базы данных (MSSQL, PostgreSQL) |
| Фронтенд | HTML5/CSS3 | HMI/SCADA-интерфейс |
| API | REST (JSON) | Обмен данными между подсистемами АСУ |

## 4. Описание основного функционала
| Функция | Описание | Аналог в АСУ ТП |
|---------|----------|------------------|
| Добавление объекта | Ввод продукта с параметрами (название, кол-во, срок) | Конфигурирование технологического объекта |
| Хранение данных | SQLite-база с таблицей продуктов | База данных параметров процесса |
| Автоматический контроль | Расчёт статуса: "Годен"/"Истекает"/"Просрочен" | Мониторинг предельных значений (аварийная сигнализация) |
| Визуализация | Цветовая индикация (зелёный/жёлтый/красный) | Цветовое кодирование на мнемосхемах SCADA |
| Статистика | Количество годных/просроченных продуктов | Сводные отчёты по технологическим показателям |
| REST API | `/api/stats` — выдача статистики в JSON | Интеграция с внешними системами (MES, ERP) |
